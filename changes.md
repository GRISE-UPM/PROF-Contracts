# Document here the issues associated to items g) and h)
Respecto al g, para poder probar con un valor negativo de fee sólo se puede hacer, sin tocar el código original, con un máster, y no con un grado. Se usa un Ensures en el constructor de cada tipo de Course, en este caso, pues es donde me parece más adecuada la condición.

Respecto al h, surge el problema de que si el g ya comprueba que no existan asignaturas negativas, rellenar una matrícula con asignaturas negativas para probar que esta no pueda ser tampoco negativa no es posible sin tocar el código. Quizá se podría solucionar usando mocking, pero como no es algo concerniente a este ejercicio lo he desechado. Lo que se me ha ocurrido para lograr probar este test, sin tocar el código original, es crear un nuevo tipo de Course, NegativeCourse, que siempre devuelva un fee de -1. Este test está en el package de prueba, y es con el que se rellena una matrícula negativa para comprobar que debe saltar exceipción. En este caso se usa un Invariant por no haber un constructor más que el de por defecto en Registration.
